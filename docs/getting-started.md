# 快速开始

## 1. 先启动服务

BatteryRecorder 依赖独立的服务进程采集数据。  
如果首页还没有连接服务，先按首页提示选择启动方式：

- Root 启动
- ADB 启动（推荐使用nohup版本）

## 2. 进入首页确认正在记录

服务启动后，首页会出现这些信息：

- 当前记录
- 充电总结 / 放电总结
- 续航预测卡片

> 如果你当前处于充电状态，续航预测入口不会显示。

## 3. 必须的操作

进入“设置”后，建议先确认这些分组里的项目：

- “常规”里的校准相关项目，见 [校准说明](calibration.md)
- “预测”里的高负载 App 排除项，见 [排除高负载 App](high-load-apps.md)

> 前者决定**功率显示**是否正常，后者保证**续航预测**准确。

## 4. 继续往下看什么

- 想调整显示：看 [校准说明](calibration.md)
- 想排除游戏或其他高负载应用：看 [排除高负载 App](high-load-apps.md)
- 想调整采样、写入或分段：去设置页的“服务”分组
- 如果息屏判断异常：去设置页的“服务”分组开启“轮询获取息屏状态”
- 想调整日志保留天数或日志级别：去设置页的“日志”分组
- 想看预测结果怎么解读：看 [续航预测说明](prediction.md)
- 想管理历史文件：看 [历史记录操作](history.md)
- 想看图表交互：看 [记录详情图表说明](record-detail.md)
